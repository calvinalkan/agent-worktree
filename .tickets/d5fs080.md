# Add --json flag to wt create command

The wt create command should support a --json flag to output the new worktree info as JSON instead of human-readable text.

Current output (lines 214-220):
  Created worktree:
    name:        <name>
    agent_id:    <agent_id>
    id:          <id>
    path:        <path>
    branch:      <branch>
    from:        <base_branch>

With --json flag, output should be:
{
  "name": "<name>",
  "agent_id": "<agent_id>",
  "id": <id>,
  "path": "<path>",
  "branch": "<branch>",
  "from": "<base_branch>"
}

This enables scripting and tool integration.

## Acceptance Criteria

- Add --json flag to create command flags (line 26-29)
- When --json is set, output JSON object with all worktree info
- JSON output should go to stdout, any warnings still to stderr
- Existing human-readable output remains default behavior
